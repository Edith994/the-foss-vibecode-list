# Contributing to The FOSS&Vibecode List

Thanks for contributing!

This repository documents software across two development cultures:

- **foss.md** → traditional open-source software developed primarily through conventional human-led workflows
- **vibe.md** → projects where AI assistance, LLM-native workflows, or vibe-coding materially shaped development

## Submission Guidelines

When proposing a project, please include:

- Project name
- Official website
- Source repository
- Short description
- Why it belongs in this category

## Inclusion Criteria

### foss.md
Projects generally belong here if they are:
- Open source
- Maintained through traditional engineering workflows
- Human-led in architecture and implementation

### vibe.md
Projects generally belong here if:
- AI materially accelerated development
- Maintainers openly discuss AI-assisted coding
- LLM tools significantly influenced implementation or architecture

## Ambiguous Cases

Some projects may fit both categories or sit between them.

When in doubt:
- explain your reasoning
- link supporting evidence if available

## Formatting

Please follow the existing Markdown format used in the repository.

## Respect the spirit of the list

The goal is thoughtful classification, not strict purity.
Reasonable disagreement is expected.
